  • s-(n) est disponible pour tout les formats si il n'y a pas d'autre taille spécifié "l-(n), m-(n))"
  • m-(n) et l-(n) sont wrapper dans des medias query et donc disponible uniquement dans leur tailles respective, par exemple .m-(n) est inactive en large et small
  • Offset : o-(n) va déplacer une div de (n) block par la gauche, est disponible pour toutes les tailles

Responsive utilities

  • .h-o-s, .h-o-m, .h-o-s => hide offset for small, medium, large. Si non spécifié l'offset sera disponible pour toute les tailles
  • .h-s, .h-m, .h-l => hide for small, medium, large.

Modal

La modal prend en compte l'id de la modal et une fonction js.
A vous de voir si vous voulez laisser l'event dans l'html ou le déplacé dans votre fichier .js.

<div class="modal center" id="test-modal"> <h3>Shibe of the year</h3> <img src="images/doge.jpeg" alt="" width="200"/> <p> Lorem... </p> <div class="btn default" onCLick="akakitModal('test-modal')">Fermer</div> </div> <button class="btn primary" onCLick="akakitModal('test-modal')">Start</button>

La modal est basic, niveau js on a que quelques lignes :

akakitModal = (id) => { var modal = document.getElementById(id).classList; var overlay = document.getElementsByClassName('modal-overlay')[0]; overlay.onclick = () => { overlay.classList.remove('active'); modal.remove('active'); }; if(modal.contains("active")){ overlay.classList.remove('active'); modal.remove('active'); }else{ overlay.classList.add('active'); modal.add('active'); } };
